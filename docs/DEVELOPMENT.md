## For Developers working on project:

1. Forking `main`
   - Clone `main` branch into your own fork
   - Branch name should be `[STUDENT NUMBER]-[FIRST LETTER OF NAME][FIRST LETTER OF SURNAME]`
   - For example: 220483418-CS
2. Development
   - If necessary, add the files which are attached to the issue assigned to you to your project. This is mainly skeleton-code to fix missing dependencies since we are working at different files that are interlinked
   - Add the following comment (using your own details) at the top of each file you create:
     ```java
         /*
         User.java
         User model class
         Author: Charlton Solomons (220483418)
         Date: 11 March 2026
         */
     ```
3. Uploading your code
   - Before committing, update your branch with the latest changes from `main`:
        ```
        git pull origin main
        ```
   - Only add the files which you changed
   - Commit your changes
   - Push to your branch
4. Pull request
   - From your own branch, create a pull request to the **Development** branch
5. Clean-up
   - If your code needs adjustments, a comment will be left on the pull request and / or the issue assigned to you
   - Once your code is approved, it will be merged into `Development`

**Quick note:**

- Once all code is in `Development`, it will be merged to main.
